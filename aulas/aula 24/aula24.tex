\section{Divisão e Conquista}
\begin{frame}[fragile]
\frametitle{Divisão e Conquista}

\begin{itemize}
\item \alert{Dividir} o problema em um certo número de subproblemas;
\item \alert{Conquistar} os subproblemas resolvendo-os recursivamente. Se o tamanho 
do subproblema é suficientemente pequeno (caso base) resolva-o diretamente.
\item \alert{Combine} as solu\c{c}ões dos subproblemas para encontrar uma solu\c{c}ão do 
problema original.
\end{itemize}

\end{frame}

\subsection{Exponencia\c{c}ão binária}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Exponencia\c{c}ão binária}
Podemos utilizar esta abordagem para definir a potência um outro algoritmo de exponencia\c{c}ão.\\

Neste caso, $x^n$ é calculado:
\begin{itemize}
\item Caso  básico: 
  	\begin{itemize}
  	\item Se $n=0$ então $x^n = 1$.
  	\end{itemize}
\pause
\item Caso Geral:
  	\begin{itemize}
  	\item Se $n>0$ e é par, então $x^n = (x^{n/2})^2$.
  	\item Se $n>0$ e é ímpar, então $x^n = x(x^{(n-1)/2})^2$.
  	\end{itemize}
\end{itemize}
Note como no caso geral definimos a solução do caso maior em termos
de casos menores.
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Exponencia\c{c}ão binária}
Este algoritmo é mais eficiente do que o clássico. Por que?
Quantas chamadas recursivas o algoritmo pode fazer?
\begin{small}
\begin{verbatim}
double power(double a, int n)
{
   double m;
   //Caso base
   if(n == 0) 
       return 1;
   if(n == 1) 
       return a;
\end{verbatim}
\end{small}
\pause
\begin{small}
\begin{verbatim}
   m = power2(a, n/2);
   
   if(n % 2)
       return m*m*a;
   else 
       return m*m;		
}
\end{verbatim}
\end{small}
\end{frame}

\begin{frame}[fragile]
\frametitle{Exponencia\c{c}ão: Algoritmo clássico}

\begin{verbatim}
double power(double a, int n)
{
   if(n == 0) return 1;
   if(n == 1) return a;
   return a*power(a, n-1);
}
\end{verbatim}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Exponencia\c{c}ão binária}
\begin{itemize}
\item Na exponencia\c{c}ão binária, a cada chamada recursiva o valor de $n$
é dividido por 2. Ou seja, a cada chamada recursiva, o valor de $n$
decai para pelo menos a metade. 

\item Usando divisões inteiras faremos
no máximo $\lceil (\log_2 n) \rceil +1$ chamadas recursivas.

\item Enquanto isso, o algoritmo clássico executa faz $n$ multiplica\c{c}ões. 
\end{itemize}
\end{frame}

int buscaBinaria(int v[], int inicio, int fim, int chave)
{
    int m = (inicio+fim)/2;	
	
    if(inicio > fim) return 0;
    if(inicio == fim) return  (v[m] == chave);
    
    if(chave == v[m])
        return 1;
    if(chave < v[m])
        return buscaBinaria(v, inicio, m-1, chave);
    else
        return buscaBinaria(v, m+1, fim, chave);

}

\begin{frame}[fragile]
\frametitle{Recursão com várias chamadas}
\begin{itemize}
\item Não há necessidade da função recursiva ter apenas
uma chamada para si própria.
\item A função pode fazer várias chamadas para si própria.
\item A função pode  ainda fazer chamadas recursivas indiretas. Neste caso
a função 1, por exemplo, chama uma outra função 2 que por sua vez chama
a função 1.  
\end{itemize}
\end{frame}


\subsection{Soma de um Vetor}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Soma de um vetor}
\begin{itemize}
\item Dado um vetor, vamos definir $S(i,n)$ como a soma de $n$ elementos a partir da posição
$i$. 
\item Com isso temos a seguinte definição
recursiva para a soma dos elementos de um vetor:
	\begin{itemize}
	\item Se $n=1$ então $S(i,n) = v[i]$.
	\pause
	\item Se $n>1$ então $S(i,n) = S(i,\lceil n/2 \rceil) + S(i+\lceil n/2 \rceil, \lfloor n/2 \rfloor)$.
	\end{itemize}
\pause
\item Observações
	\begin{itemize}
		\item $n = \lceil n/2 \rceil + \lfloor n/2 \rfloor$.
		\item Dada uma posição $i$ e quantidade $x= \lceil n/2 \rceil$ de elementos incluindo $x$, a próxima posição
		a ser considerada será $(i+x)$.
	\end{itemize}
\item Para computarmos a soma de todos os elementos de um vetor com $n$ elementos, devemos calcular $S(0,n)$.

\end{itemize}

\end{frame}

\begin{frame}[fragile]

int soma(int v[], int i, int n){
  if(n == 1)
    return v[i];
  else{
    return soma(v, i, teto(n,2)) +
      soma(v, i+teto(n,2), chao(n,2));
  }
}
\end{verbatim}

\begin{frame}[fragile]
\frametitle{Torres de Hanoi}
\begin{itemize}
\item Inicialmente temos 5 discos de diâmetros diferentes na estaca A.
\item O problema das torres de Hanoi consiste em transferir os cinco discos
da estaca A para a estaca C (pode-se usar a estaca B como auxiliar).
\item Porém deve-se respeitar algumas regras:
\begin{itemize}
\item Apenas o disco do topo de uma estaca pode ser movido.
\item Nunca um disco de diâmetro maior pode ficar sobre um disco de diâmetro menor.
\end{itemize}
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Torres de Hanoi}
\begin{itemize}
\item Vamos considerar o problema geral onde há $n$ discos.
\item Vamos usar indução para obtermos um algoritmo para este
problema.
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Torres de Hanoi}
\begin{teorema} É possível resolver o problema das torres de
Hanoi com $n$ discos.
\end{teorema}
\begin{prova}
\begin{itemize}
\pause
\item Base da Indução: $n=1$. Neste caso temos apenas um disco. Basta
mover este disco da estaca A para a estaca C.
\pause
\item Hipótese de Indução: Sabemos como resolver o problema quando há
$n-1$ discos.
\end{itemize}
\end{prova}
\end{frame}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Torres de Hanoi}
\begin{prova}
\begin{itemize}
\item Passo de Indução: Devemos resolver o problema para $n$ discos assumindo
que sabemos resolver o problema com $n-1$ discos.
\pause
\item Por hipótese de indução sabemos mover os $n-1$ primeiros discos
da estaca A para a estaca B usando a estaca C como auxiliar.
\pause
\item Depois de movermos estes $n-1$ discos, movemos o maior disco (que
continua na estaca A) para a estaca C.
\pause
\item Novamente pela hipótese de indução sabemos mover os $n-1$ discos
da estaca B para a estaca C usando a estaca A como auxiliar. \\

\item Com isso temos uma solução para o caso onde há $n$ discos.
\end{itemize}
\end{prova}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Torres de Hanoi: Passo de Indução}
\begin{itemize}
\item Olhem que maravilha que é a indução.
\item Nos fornece um algoritmo e ainda por cima temos
uma demonstração formal de que ele funciona!
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}[fragile]
\frametitle{Torres de Hanoi: Algoritmo}
Problema: Mover $n$ discos de A para C.
\begin{enumerate}
\item Se $n=1$ então mova o único disco de A para C e pare.
\item Caso contrário ($n>1$) desloque de forma recursiva os $n-1$ primeiros discos de A para B,
usando C como auxiliar.
\item Mova o último disco de A para C.
\item Mova, de forma recursiva, os $n-1$ discos de B para C, usando A como  auxiliar.
\end{enumerate}
\end{frame}
